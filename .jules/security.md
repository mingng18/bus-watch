## 2024-12-16 - Timing Attack vulnerability fix
**Learning:** Hono's `timingSafeEqualString` built-in function returns false gracefully when lengths mismatch (rather than throwing), however standard practice and core library requirements often enforce length-checks before passing strings to `timingSafeEqual` in Node.js environments. By dynamically tracking token lengths beforehand and conditionally ensuring the length is the same against a fallback comparison token, length leaks and fail-open vulnerabilities can be fully remediated inside manually created auth handlers.
**Action:** Enforced consistent length checks explicitly and passed sanitized token inputs to `timingSafeEqual` inside Hono route handlers to eliminate any token-length side-channels during Authorization header evaluation.
