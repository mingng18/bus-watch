## 2026-05-24 - Optimize Geo-Proximity Lookups by Pre-Computing GTFS Hash Maps
**Learning:** Geospatial scans (like finding nearby buses and stops) are highly active endpoints. In `findNearbyStops`, `findNearbyBusRoutes`, and `findNearbyPrasaranaBuses`, there were hidden O(N) penalties inside O(N) loops. Specifically, relational GTFS model links (e.g. searching an entire `trips` array inside a loop of nearby vehicles) compounded into an O(N^2) issue. Replacing `trips.find` with an O(1) `Map` instantiation before the loop dramatically cuts processing time and makes the function O(N) scale.
**Action:** When performing searches or filtering over geographic arrays (stops, vehicles, routes), always verify that relation lookups on parent data (like looking up a route or trip for a vehicle) are done via pre-constructed O(1) hash maps instead of embedded `Array.prototype.find()` iterations.
