## 2024-05-19 - Avoid Chained Array Iterations for Critical Paths
**Learning:** Chaining `.filter().map()` incurs an intermediate array allocation which degraded performance inside the hot path `index.ts`.
**Action:** Replaced `.filter().map()` loops with highly optimized classic `for` loops containing precomputed property accesses (`routeId` / `routeShortName`), which yielded measurable execution speedups.
## 2023-10-27 - Fast GTFS time string parsing without allocating arrays
**Learning:** Parsing times like `12:34:56` or `12:34` in tight loops using `.split(':').map(Number)` allocates a significant number of intermediate objects, leading to measurable performance degradation when processing thousands of schedules. However, replacing it with `indexOf` and `substring` without careful consideration of missing seconds (e.g. `12:34` instead of `12:34:00`) introduces functional regressions because GTFS allows omitting seconds. Attempting string manipulation for zero-padding also introduces errors. Micro-optimizations of strings often sacrifice readability and introduce bugs. Array iteration optimizations like avoiding `.map().filter()` are much safer.
**Action:** Do not sacrifice readable string parsing like `.split(':')` for `indexOf`/`substring` micro-optimizations, as they are error-prone and violate Bolt's boundary of readability over micro-optimizations. Focus strictly on replacing array mapping/filtering chains with `for` loops which provide solid, safe performance gains.
